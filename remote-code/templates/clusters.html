There are two methods of clustering presented in this section: k-means and hierarchical clustering. The hierarchical clustering algorithm is presented as a dendrogram where the user can approximate how related two skills are(we have also emphasized the most important clusters). The k-means algorithm was run for a number of clusters that varied between 10 and 16.

{% endblock %} {% block text_tech %}

We have presented two clustering methods applied on our skills set. Before feeding the data to the clustering algorithms we have computed the feature vectors for each skill This feature vector is a binary vector where an entry represents whether a project requires that skill.

The k-means clustering algorithm divides the data into clusters based on the number of clusters decided by the user. It is a fast iterative algorithm which tries to gather the data around central points of the space: centroids. We have run the k-means algorithm using between 10 and 16 clusters. It can be seen that the one with X clusters intuitively makes more sense because it gathers only the skills that go together

The hierarchical clustering algorithm takes longer, but the clusters are more accurate. The hierarchical clustering results are presented in a dendrogram, which also shows the links between clusters. This way, the user can see by the actual length of the links how close the clusters are. The most important clusters have been marked directly on the dendrogram.

{% endblock %} {% block text_analysis %}

As it can be seen from both clustering methods, the data can be easily put into clusters that make sense. It can be seen in the dendrogram that all the selected clusters are actually related to one another from a technical perspective. For example, all the operating systems are gather together, as well as most of the common programming languages
